# Contribution Guidelines

Thank you for your interest in contributing to FileDonkey! We welcome contributions from everyone. Please follow the guidelines below to make your contribution process smooth.

## Getting Started
1. **Fork the Repository**: Click the "Fork" button on the top right to create your personal copy of the repo.
2. **Clone Your Fork**: Clone your forked version to your local machine:
   ```bash
   git clone https://github.com/YOUR_USERNAME/filedonkey.git
   ```

3. **Create a Branch**: Create a new branch for your feature or bug fix:
   ```bash
   git checkout -b your-feature-name
   ```

## Making Changes
- Ensure your changes are made in the new branch you created.
- Write clear, concise commit messages that describe your changes.

## Submitting Your Contribution
1. **Push Your Changes**: Push your changes to your forked repository:
   ```bash
   git push origin your-feature-name
   ```
2. **Create a Pull Request**: Go to the original repository and click the "New Pull Request" button.
   Make sure to select your branch from the dropdown list.

## Code of Conduct
All contributors are expected to adhere to our [Code of Conduct](CODE_OF_CONDUCT.md).

## Thank You!
We appreciate your contributions and are excited to work together!
